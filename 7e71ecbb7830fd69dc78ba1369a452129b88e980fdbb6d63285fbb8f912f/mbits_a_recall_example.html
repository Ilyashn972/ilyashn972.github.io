Calculate meaningful bits for a recall clause vs all the previous ones

The plots below illustrate how recall clauses influence meaningful bits (which are actually computed as the sum of *natural* log probabilities ln ).

Each row represents a clause, denoted as Ci. The cell colors indicate the difference in probabilities for the corresponding segments, obtained from two separate probability calculations:

  1. Using all clauses from C1 to Ci-1.
  2. Using all clauses from C1 to Ci.

Black dots represent "caught" segments, while white circles mark "recalled" segments. Segments that were "recalled" but not "caught" appear in the top row, which serves as a placeholder (xa xa).
