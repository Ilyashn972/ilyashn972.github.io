7. Scripts involved

How these results were produced:

  • Correlation CSV: tools/build_pcaught_emulation_correlation_csv.py — builds story×model correlation from human vs emulated p-caught (reads clauses_to_segments_set.csv, uses tools.get_counts_of_caught). Output rounded to 3 decimal places.
  • Self-correlation: Table is computed inline by this report using experiments.pcaught_self_correlation_html.compute_stats_for_story (50/50 split of recalls, correlation between halves).
  • Pipeline stats: tools/build_recall_pipeline_stats_csv.py — counts per story: recalls in JSON (emulated), segmented, and mapped. Output: html/recall_pipeline_stats_by_story.csv.
  • Chars per recall: Computed inline from story JSON recalls (character count of each recall text); mean and std for human and each emulation model.
  • Per-story plots: This report script generates Plotly HTML per story (human + emulation series) under html/reports/plots/; logic mirrors experiments/plot_pcaught_comparison.py.
  • Data pipeline: bin/run_recall_emulation_orchestrator.py (emulate_recall → build_json_from_new_files → map_to_segments), experiments/emulate_recall.py, key_point_tree/.../map_to_segments.py (produces clauses_to_segments_set.csv).
