Comprehensive Leave-One-Out Report: Costa MBits

This report consolidates the leave-one-out MBits analysis for Costa into one canonical document. It links all currently available leave-one-out plot variants, explains the pipeline and prompts, and records reproducibility details for reruns and extensions.

Quick links: all | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 | 1000 | 1175

1) Scope and research question

The leave-one-out analysis asks: if one summary is held out, how do MBits change when comparing that held-out summary against other summaries and against matched alternatives (recall, extractive Doron summary, and random extractive Doron summary), including original and rephrased variants?

MBits here are computed as sums of token log-probability deltas through tools/get_mbits.py, contrasting narrative probability with and without a preceding text context.

2) Main assets and scripts

  • Main generator: experiments/mbits_summary_leave_one_out.py
  • Core MBits function: tools/get_mbits.py
  • Detailed single-run explanation page: mbits_costa_summary_leave_one_out_5_explanation.html

3) Pipeline overview

  1. Load Costa clauses and summary cache from processed data/cache paths in the script.
  2. Choose an excluded summary (leave-one-out target) based on requested target_length logic.
  3. Select length-matched comparators:
    • human recall closest in character count,
    • extractive Doron summary closest in character count,
    • random extractive Doron summary with similar length.
  4. Compute MBits for multiple modes in parallel (summary mode, recall mode, Doron mode, random Doron mode).
  5. Compute additional “narrative/rephrased narrative after X” anchor points.
  6. Render interactive Plotly output and write: html/costa/mbits_costa_summary_leave_one_out_{target_length}.html.

4) Prompt and rephrasing context

The leave-one-out flow uses both original and rephrased text variants to separate content effects from lexical/syntactic wording effects. Rephrasing prompts and the narrative rephrasing strategy are documented on: the dedicated explanation page.

6) How to read the plots

  • X-axis: text length (character-based in this analysis context).
  • Y-axis: summed MBits/log-probability effect under the selected comparison mode.
  • Markers separate original vs rephrased variants (circles vs crosses in the documented reference run).
  • Traces are grouped by mode (summary, recall, extractive Doron, random extractive Doron), with narrative anchors.
  • The held-out summary defines the target context for leave-one-out in the given run.

7) Reproducibility

From project root, run: 

python experiments/mbits_summary_leave_one_out.py costa 5
python experiments/mbits_summary_leave_one_out.py costa 8
python experiments/mbits_summary_leave_one_out.py costa 1000

Output convention: 

html/costa/mbits_costa_summary_leave_one_out_{target_length}.html
html/costa/mbits_costa_summary_leave_one_out_{target_length}.png

The script defaults to: story=costa and target_length=1000 when no CLI args are provided.

8) Interpretation notes

  • Consistency across nearby target lengths (e.g., 5–9) is useful as a robustness signal.
  • Divergence between original and rephrased traces can indicate wording sensitivity rather than content sensitivity.
  • Comparison to random extractive Doron acts as a control for structure-matched but semantically looser inputs.
  • Long-target runs (1000, 1175) can reveal scale effects that are not visible in short keypoint-like variants.

9) Limitations and threats to validity

  • Token-logprob derived MBits can vary by backend/model version and API behavior.
  • Length matching is approximate and may introduce residual confounding.
  • Rephrasing quality can differ by run/model response variability, affecting downstream MBits.
  • Single-story focus (Costa) limits direct generalization without repeating the protocol on additional stories.
