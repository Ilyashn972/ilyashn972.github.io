Reports

This page lists the main reports that describe the Story Recall Research work. Each report is linked below with a short description of its contents.

Narrative and summary reports

  • Again report (delayed recall)

    Narrative summary of delayed-recall experiments: methodology (text-only after one month, multimedia after one week), average c and n, fan-out and intrusions, the “orange” recall finding, and LLM split-and-map (GPT-5.2) with compression ratio. Links to the full again/experiments page.

  • Again folder inventory

    Index of everything in again/: entrypoints, counts by file type, largest/recent files, CSV-backed table-view symlinks, and per-subfolder listings with links.

  • Story pairs report

    Analysis of p-caught and p-rec correlation across story pairs (Schissel, Triplett, Hester). Method, scatter plots, and finding that there is no correlation for Hester. Links to story_pairs plots.

  • Model comparison report

    Comparison of GPT-4o vs GPT-5.2 and of different prompts for p-caught correlation with human recall. Main findings: prompt matters less than model; GPT-4o is more human-like than GPT-5.2. Notes on temperature and peak recall. Links to Costa p-caught comparison explanation.

  • Costa multimedia report

    Documentation of the Costa multimedia dataset: purpose, first illustration attempt (Jul 2025), limitations (TTS/STT, consistency). Links to review page, interactive story, and dataset zip.

Meaningful bits

  • Comprehensive leave-one-out report (Costa MBits)

    Canonical leave-one-out documentation for Costa MBits: research question, methodology, prompts/rephrasing context, full plot catalog (2,3,4,5,6,7,8,9,1000,1175,all), reproducibility commands, interpretation notes, and limitations.

  • Mbits report

    Results related to meaningful bits (difference in sum of log-probabilities with vs without recall as preceding context). Includes prompts, same-subject mbits plots, recall-vs-summaries plots, and links to various mbits visualizations and code.

Emulation and recall simulation

  • Emulation report

    Toward emulating human recall: LLM-based emulation (keypoints vs clauses, correlation vs GPT-4o max clause id, heatmaps, etc.) and links to plots grouped by story. Compares emulated vs real recall patterns.

  • Emulation by Robot report

    Model comparison table for emulating a person by a robot: different models and methods with thumbnails and links to detailed plots and data.

  • Recall emulation report

    Main recall-emulation report: Pearson correlation (story × model) of human vs model p-caught for GPT-4o, Llama, DeepSeek, GPT-OSS-120B; self-correlation; text length stats; segment- and keypoint-level tables; links to CSV-backed tables and scripts.

  • Recall emulation plots

    Per-story p-caught comparison: one plot per story (schissel, boyscout, triplett, hester, panic, doSomething, bear, stein, costa). Human vs model p-caught. Links to individual plot pages; some views use iframes.

  • Correlation bar

    Bar chart of story × model correlations (human p-caught vs each model’s p-caught) across stories, for quick comparison of how well each model matches human recall.

Aggregate tables

  • All stories p-caught table

    Interactive, sortable table of segment-level p-caught across all stories: one row per (story, segment_id) with human and model p-caught, model mean/min/max, and mean_minus_human. Sorted so segments where models diverge most from humans are easy to find.

  • All stories keypoints table

    Interactive, sortable table of keypoint-level (clause) p-caught across all stories: one row per keypoint with aggregated human and model p-caught and mean_minus_human. For finding key ideas where models under- or over-catch relative to humans.
