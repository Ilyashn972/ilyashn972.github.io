Heatmap emulation by gpt4 7 keypoints

Prompt: 
                    Role: Emulate a person who can remember only 7 key points at a time.
                    
                    Task:
                    You will receive a narrative, provided in separate paragraphs over multiple turns. At each turn, the user will give a
                    new paragraph of the narrative. Your job is to maintain a running summary consisting of up to 7 key points derived from
                    all paragraphs received so far, including the new one. Each key point must:
                    • Be a short linguistic clause.
                    • Contain exactly one verb.
                    • Summarize information from the narrative without interpreting it.
                    • Be added only if the paragraph introduces new information.
                    • Not repeat previously stated information, even if the narrative does.
                    • May be modified for clarity or completeness as the narrative evolves.
                    
                    The output must be returned in JSON format:
                    ```json
                    { "keypoints":[
                    "First key point",
                    "Second key point",
                    // and so on...
                    
                    ]}```
                    You may include fewer than 7 key points, but never more than 7.

Each line in the plot corresponds to a new paragraph. The vertical bar at the end of each line shows which segments the LLM has summarized. The colors represent the compression ratio—that is, how many segments are included in the corresponding fan_out.

The results were noisy, unfortunately. Due to some randomness in GPT-4, it doesn’t always split the keypoints into the same clauses, and it doesn’t consistently map the same clauses to the same segments. So I have decided not to split into clauses. If the same keypoint is repeated and mapped to different segments, I'll merge all mappings into one. Of course I will limit the mappings to the segments that were fed to the LLM at that time.

The final mappings are not very similar to those of a typical human subject. :(

The last paragraph of a narrative has a strong influence on the mappings—possibly due to its reflective nature.

In Stein’s story, you can see that as the narrative progresses, the same subject performs an extractive summary at the beginning and an abstractive summary at the end.

Can we consider a stochastic process for keypoint dynamics under memory constraints? For example, when memory is limited and new information arrives, a few strategies might occur:

  •  A. Ignore it (perhaps it’s deemed unimportant)
  •  B. Merge it with an already existing keypoint
  •  C. Merge two older keypoints and add the new one
  •  D. Remove or demote an older keypoint to make room for the new one
  •  E. Recall one of the previous keypoints and merge it with the new one.
